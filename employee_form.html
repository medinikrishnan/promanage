User Schema Definition

The user schema for our database has been fully defined. This schema is used to manage user information in our system and follows best practices for relational database design.

Table: Users

  • user_id: Primary Key, INT, auto-incremented, not null
  • username: VARCHAR(255), unique, not null
  • email: VARCHAR(255), unique, not null
  • password: VARCHAR(255), not null
  • createdAt: DATETIME, defaults to current_timestamp
  • updatedAt: DATETIME, defaults to current_timestamp, automatically updates on record change

This schema has been implemented in our models and reflects all required attributes for the user domain.
